## 1. 스케줄 요일 설정형(체크식) 전환

- **현재**
  - 스케줄 모달에서 `days`를 텍스트(`daily`, `Mon,Tue,Wed` 등)로 직접 입력
  - 오타 가능성이 있고, 눈으로 보기에도 직관적이지 않음
- **변경 후 (계획)**
  - 모달에 다음 UI 추가
    - `매일 (daily)` 체크박스 1개
    - `Mon / Tue / Wed / Thu / Fri / Sat / Sun` 7개 요일 체크박스
  - 동작 규칙 (예시)
    - `매일` 체크 시 → `days = "daily"`
    - `매일` 해제 후 Mon,Wed,Fri만 체크 시 → `days = "Mon,Wed,Fri"`
    - 아무 것도 선택 안 되면 저장 불가 또는 `daily`로 보정 (UX 결정 필요)
  - 저장 시
    - 기존 백엔드와 호환되도록 `daily` 또는 `Mon,Tue,...` 문자열로 변환 후 API 호출
  - 수정 예상 파일
    - `pi/templates/index.html` (모달 마크업 변경)
    - `pi/static/app.js` (체크박스 ↔ days 문자열 변환, 검증 로직)
    - `pi/static/style.css` (요일 체크박스 레이아웃)

## 2. RS485 NPK 값 웹 연동

- **센서**: Halisense TH-EC-PH-NPK (7-in-1, RS485 Modbus)
- **현재**
  - 아두이노는 온도·습도·EC·pH만 JSON으로 송신
  - 웹(프론트)는 `soil_N` / `soil_P` / `soil_K` 필드를 이미 파싱·표시하도록 준비되어 있음
- **변경 후 (계획)**
  - Halisense 매뉴얼 기준 N·P·K 레지스터 주소(예: 4/5/6 등)를 확인
  - 아두이노 `smart_farm_relay.ino`에 N·P·K 레지스터 읽기 추가
  - BADGE JSON에 다음 필드 포함
    - `"soil_N": <nitrogen_value>`
    - `"soil_P": <phosphorus_value>`
    - `"soil_K": <potassium_value>`
  - Pi/Web 쪽은 기존 로직(8차 기준)으로 그대로 수신·저장·표시 가능하므로, 실제 값이 올바르게 들어오는지만 검증
  - 수정 예상 파일
    - `arduino/smart_farm_relay/smart_farm_relay.ino`
    - (필요 시) `PATCH.md`, `patch/09.md` 갱신

## 3. 카메라 + Google Drive 업로드 양성화

- **현재 구조 (camera_project)**
  - 위치: `/home/pi/camera_project/camera_capture.py`
  - 촬영
    - `rpicam-still` CLI 사용 (`--timeout 2000`, `--width 1920`, `--height 1080`, `--shutter 12000`, `--gain 1.3`, `--awb fluorescent`, `--contrast 1.1`, `--saturation 1.15`, `--denoise cdn_off` 등)
    - 로컬 저장 경로: `LOCAL_SAVE_DIR = "/home/pi/camera_project/photos"`
    - 파일명: `YYYY-mm-dd_HH-MM-SS.jpg` 형식
  - 업로드
    - `upload_to_drive(file_path)`에서 `rclone` 실행
    - remote 이름: `ANJAEHUN_SMART_FARM`
    - 원격 폴더: `SmartFarmPhotos`
    - 명령 예: `rclone copy /home/pi/camera_project/photos/2026-03-12_16-40-00.jpg ANJAEHUN_SMART_FARM:/SmartFarmPhotos`
    - 업로드 **성공 시 해당 로컬 파일을 즉시 삭제**하여 Pi의 저장 공간 사용을 최소화 (업로드 실패 시에는 파일 유지)
  - 스케줄
    - `pi` 유저 crontab: `*/10 * * * * python3 /home/pi/camera_project/camera_capture.py`
    - → 10분마다 촬영 + rclone 업로드
- **9차에서 할 일 (문서화·정리 중심)**
  - SmartFarm 리포지토리의 `patch/09.md` 및 `PATCH.md`에 위 구조를 정식 기능으로 기록
  - 필요 시 README/설치 가이드에
    - rclone 설치 및 remote(`ANJAEHUN_SMART_FARM`) 설정 방법
    - crontab 설정 예시
    - 업로드 확인 방법
    를 추가하는 것을 검토
  - (추가 옵션) `config.json`의 `camera` 섹션과 연동하여
    - interval_seconds
    - save_dir
    등을 camera_project 쪽에서 참고할 수 있도록 구조 통일 검토

## 4. 웹 표시 개선 (카메라 상태 + 스케줄 가독성)

- **카메라 촬영/업로드 상태 표시**
  - 토양 센서 카드 영역과 스케줄 표 사이에 카메라 상태를 보여주는 영역 추가
    - 예: `00:00 촬영 및 드라이브 업로드 완료`, `00:10 촬영 실패`, `00:20 업로드 실패` 등
  - 기준 정보
    - `/home/pi/camera_project/camera_capture.py` 실행 결과(또는 로그)를 간단히 노출하는 API/표시 방식 설계
    - 최근 한 번 또는 최근 N회 결과를 보여줄지 범위 결정
  - 수정 예상 파일
    - `pi/app.py` (카메라 상태 조회 API가 필요하다면)
    - `pi/templates/index.html` (카메라 상태 섹션 마크업 추가)
    - `pi/static/app.js` (상태 조회 및 표시 로직)
    - `pi/static/style.css` (상태 텍스트 스타일)

- **스케줄 표 X축(시간/날짜) 가독성 개선**
  - 현재: 시간/날짜 텍스트가 줄바꿈되거나 좁게 표시되어 가독성이 떨어짐
  - 개선 방향
    - 시간과 날짜를 한 줄에 표시하도록 줄바꿈/공백 정리
    - 글자 크기(Font size)와 간격(Margin/Padding) 조정으로 읽기 쉬운 크기로 확대
  - 수정 예상 파일
    - `pi/templates/index.html` (시간/날짜 텍스트 구조 확인 후 필요 시 마크업 조정)
    - `pi/static/style.css` (폰트 크기, 줄 간격, 칼럼 폭 등 조정)
