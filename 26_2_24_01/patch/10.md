# 10차 패치 상세

- **회차**: **10차** (9차: 요일 체크박스·NPK·카메라 문서화 등 별도 진행)
- **요약**: 카메라 양성화(통합·노출, **음성 제거**), 연결/알림 개선(Gmail 제거 → 텔레그램 주기 현황), 유지보수(역추적 정리), 웹 UI 업데이트(카메라 섹션 위치 등)
- **용어**: 여기서 **음성**은 목소리가 아니라 **양성(공식·추적 가능)의 반대**, 즉 경로를 알 수 없는 결과물·음지 시스템을 뜻함. 10차에서는 이를 제거하고 양성(리포 내 공식 구조)만 유지하는 것을 목표로 함.

### 10차 추가·삭제 요약

**삭제**
- Gmail 연결 끊김 알림: `alert_email` 호출, `_check_disconnect_alert` 스케줄 job, `app.py` 의 `reset_alert_sent()` 호출.
- config 의 `alert` 섹션 전체.
- camera_project 음지 구조: 리포 외부 `/home/pi/camera_project` 및 해당 crontab 항목(배포 후 수동 제거).
- (중간 제거) 카메라 표시 문구 수정 UI 및 `custom_message`·`POST /api/camera/display`.

**추가**
- 카메라 통합: `pi/camera_capture.py` 에서 rpicam-still + 시간대별 노출, 웹 저장 노출값(`data/camera_exposure.json`) 적용, rclone 업로드·상태 기록.
- 텔레그램 주기 현황: `pi/telegram_status.py`, 10분마다 시리얼·릴레이(LED1/PUMP1… ON·OFF)·센서·카메라 메시지 전송.
- 웹 카메라 노출 제어: `GET/POST /api/camera/settings`, 낮/저녁/야간 shutter·gain·ev·awb 웹에서 수정.
- **즉시 촬영·업로드**: 웹 카메라 섹션에 릴레이 원격 ON/OFF 보드 느낌의 「촬영 및 업로드」 버튼 + `POST /api/camera/capture-once`. 주기와 관계없이 1회 촬영 후 드라이브 업로드.

### 10차 코드 적용 내역

- **pi/camera_capture.py**: rpicam-still 우선, 시간대별 노출 + 웹 저장값(`camera_exposure.json`) 적용, `get_photos_dir()`/`get_status_path()`/`load_exposure_settings()`/`save_exposure_settings()`, rclone 업로드·상태 파일 기록.
- **pi/telegram_status.py**: 신규. 주기별 작동 현황(시리얼·릴레이 LED1/PUMP1… ON·OFF·센서·카메라) 텍스트 조립 후 텔레그램 전송.
- **pi/scheduler_service.py**: Gmail 끊김 알림 job 제거, 카메라 주기 촬영 job·텔레그램 주기 전송 job 추가.
- **pi/app.py**: alert_email 참조 제거, `/api/camera/status` 통합 경로, `GET/POST /api/camera/settings`, `POST /api/camera/capture-once`(즉시 1회 촬영·업로드) 추가.
- **pi/templates/index.html**: 카메라 상태 섹션을 토양 센서 아래·릴레이 위로 이동, 즉시 촬영·업로드 버튼, 노출값(낮/저녁/야간) 편집 UI.
- **config.json / config.example.json**: `alert` 제거, `camera` 에 rclone_remote·rclone_path·delete_after_upload, `telegram` 섹션 추가.
- **배포 후 Pi에서 수동 작업**: 기존 `/home/pi/camera_project` 사용 중이었다면 crontab 에서 해당 항목 제거 후, `config.json` 의 `camera.enabled: true` 및 저장 경로·rclone 설정 후 앱 재시작.

## 1. 카메라 (양성화 · 노출 · 음성 제거)

### 1.1 camera_project 통합 및 양성화

- **현재**
  - `/home/pi/camera_project/camera_capture.py` + crontab + rclone 조합으로, SmartFarm 코드와 분리된 상태에서 10분마다 촬영·업로드 동작
  - GPT로 별도 작성된 **음성**(경로를 알 수 없는 결과물·음지) 구조로, 리포지토리에는 문서/상태 연동만 존재
- **10차 목표**
  - 카메라 스크립트를 SmartFarm 리포지토리 내로 편입 (예: `26_2_24_01/pi/camera/` 또는 `pi/camera_capture.py` 확장)하여 **양성(공식)** 구조만 유지
  - crontab·경로를 신규 구조 기준으로 정리
  - 기존 `/home/pi/camera_project` 및 옛 crontab 항목은 통합 완료 후 **삭제**하여 음성(음지) 시스템 제거
  - rclone remote(`ANJAEHUN_SMART_FARM`), 업로드 성공 시 로컬 파일 삭제 정책은 유지

### 1.2 시간대별 노출·조도 보정

- **배경**: 현재 고정 옵션이 온실/설치 환경 조도와 맞지 않아 밝기·색감 불만족
- **방향**: 현재 시각(낮/저녁/야간)에 따라 `rpicam-still` 인자 변경
  - 예: 낮(06~18시) / 저녁·새벽 / 야간 구간별로 `--shutter`, `--gain`, `--ev`, `--awb` 프로필 적용
- **적용**: 통합된 `camera_capture.py`(또는 동일 모듈)에서 시간대별 프로필 선택 후 촬영, 최종 파라미터는 본 문서에 기록

## 2. 연결 개선 (Gmail 제거 → 텔레그램 주기 현황)

### 2.1 제거

- **alert_email.py**: 시리얼 연결 끊김 시 Gmail 발송 — 시리얼이 아예 미연결일 때는 동작하지 않아 실효성 낮음 → **제거**
- **scheduler_service.py**: `_check_disconnect_alert` job 제거
- **app.py**: 시리얼 재연결 시 `alert_email.reset_alert_sent()` 호출 제거
- **config**: `alert` 섹션 중 이메일(smtp_*, to_email, disconnect_seconds) 관련 제거 또는 텔레그램용으로 교체

### 2.2 추가

- **텔레그램 주기 현황 송신**
  - 특정 주기(예: 1시간, 6시간)마다 작동 현황을 텔레그램으로 전송
  - 포함 내용(예): 시리얼 연결 여부, 최근 센서값·릴레이 상태, 카메라 최근 촬영/업로드 결과 등
  - 설정: 봇 토큰·채팅 ID는 환경변수 또는 `config.json`의 `telegram` 섹션

## 3. 유지보수 (역추적 정리)

- **목적**: GPT로 추가된 업로드 주기·촬영 관련 코드·문서를 역추적하여 정리. **음성**(경로를 알 수 없는 결과물·음지) 제거 후 **양성**(리포 내 공식 구조)만 유지.
- **정리 대상**
  - 리포 내: camera_project 하드코딩 경로 제거, 카메라/업로드 관련 API·UI를 통합 구조 기준으로 수정
  - Pi 실기: camera_project 폴더·옛 crontab 항목 제거 후, 리포 내 한 구조만 사용
- **유의**: 8차의 “10분”은 **토양 센서 BADGE 송신 주기**이므로 제거 대상 아님

## 4. 웹 UI 업데이트

### 4.1 카메라 상태 섹션 위치

- **현재**: 토양 센서 → 릴레이 → 카메라 상태 → 스케줄
- **변경 후**: 토양 센서(실시간/7일 평균) → **카메라 상태** → 릴레이 원격 ON/OFF → 스케줄
- **의도**: “센서 상태(토양·카메라) → 제어(릴레이) → 스케줄” 흐름으로 정렬
- **수정 예상**: `pi/templates/index.html` 섹션 순서, 필요 시 `pi/static/style.css`

### 4.2 카메라 상태 표시

- 통합된 카메라 구조의 저장 경로·상태 파일(또는 API) 기준으로 “최근 촬영·업로드” 문구 갱신
- `app.py`의 `/api/camera/status`가 리포 내 경로 또는 통합 스크립트 출력을 참조하도록 수정

### 4.3 즉시 촬영·업로드 버튼 (릴레이 보드 느낌)

- **추가**: 카메라 상태 섹션에 「즉시 촬영·업로드」 블록. 릴레이 원격 ON/OFF 카드와 비슷한 스타일(테두리·패딩·버튼).
- **동작**: 버튼 클릭 시 `POST /api/camera/capture-once` 호출 → 주기와 관계없이 1회 촬영 후 드라이브 업로드, 결과에 따라 카메라 상태 문구 갱신.
- **삭제**: 없음 (신규 UI·API만 추가).

## 5. 배포 후 점검 항목 (예정)

1. 카메라 상태 섹션이 토양 센서 아래, 릴레이 위에 배치되었는지 확인
2. **즉시 촬영·업로드** 버튼 클릭 시 1회 촬영·드라이브 업로드가 동작하는지 확인
3. 카메라 스크립트가 리포 내 한 구조로만 동작하는지, crontab·camera_project 정리 여부 확인
4. rclone 업로드 및 업로드 성공 시 로컬 삭제 정책이 유지되는지 확인
5. 시간대별 노출 보정(및 웹에서 수정한 노출값) 적용 시 촬영 품질 확인
6. 텔레그램 주기 현황이 설정한 주기로 전송되는지 확인
7. Gmail 알림 관련 코드·스케줄이 제거되었는지 확인
