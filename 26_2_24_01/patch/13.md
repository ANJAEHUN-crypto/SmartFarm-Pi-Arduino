# 13차 패치 상세

- **요약**: (1) 릴레이 상태 조회 주기를 낮추고 시리얼 명령을 직렬화해 **릴레이 연결 안정성**을 높임. (2) 시리얼 응답 실패 시 자동 재연결·재시도를 추가. (3) 아두이노 센서 송신 주기를 **3초**로 복원하고, 센서 읽기 중에도 릴레이 명령을 더 자주 처리하도록 개선.

---

## 1. 릴레이 상태 폴링 완화

### 1.1 문제
- 웹 UI가 릴레이 상태를 너무 자주 조회해 시리얼 명령이 과도하게 누적될 수 있었음.

### 1.2 목표
- 상태 표시의 최신성은 유지하되, 시리얼 부하를 낮춰 릴레이 응답 실패를 줄임.

### 1.3 수정
- **`pi/static/app.js`**
  - 릴레이 상태 `STATE` 폴링 주기를 `3초`에서 **30초**로 완화.
  - 시리얼 연결 상태 조회도 **30초 주기**로 갱신.
  - 페이지 진입 시 1회 즉시 상태 조회는 유지.

### 1.4 삭제
- 없음.

---

## 2. 시리얼 릴레이 자동 복구 강화

### 2.1 문제
- 릴레이 명령이 동시에 들어오거나 응답이 늦으면 `0.5초` 타임아웃으로 실패했고, 실패 후 자동 복구가 없었음.

### 2.2 목표
- 릴레이 명령을 안전하게 순차 처리하고, 응답 실패 시 자동 재연결로 복구 가능하게 함.

### 2.3 수정
- **`pi/serial_relay.py`**
  - `_io_lock` 추가로 릴레이 명령을 **한 번에 하나씩** 처리.
  - 응답 큐를 명령 전 비우고, 응답 대기 시간을 **2.5초**로 확대.
  - 실패 시 마지막 포트/보드레이트로 `close -> open` 자동 재연결 후 재시도.
  - 시리얼 오픈 후 아두이노 리셋 시간을 고려해 대기 시간을 늘림.
  - `get_state()`는 통신 예외 시 `None` 반환으로 안전 처리.
- **`pi/scheduler_service.py`**
  - 스케줄 실행 시 `is_open()` 체크로 바로 포기하지 않고, 릴레이 명령이 내부 재연결 로직을 타도록 조정.

### 2.4 삭제
- 없음.

## 3. 센서 3초 주기 복원 + 아두이노 시리얼 처리 개선

### 3.1 문제
- 센서 읽기 중에는 USB 시리얼 명령 처리가 밀릴 수 있어 릴레이 응답이 늦어질 수 있었음.
- 센서 송신 주기도 현재 요구와 맞지 않게 길어져 있었음.

### 3.2 목표
- 센서값은 **3초 주기**로 계속 받고, 센서 읽는 중에도 릴레이 명령을 더 빨리 처리하게 함.

### 3.3 수정
- **`arduino/smart_farm_relay/smart_farm_relay.ino`**
  - 센서 송신 주기를 **3초**로 복원.
  - `readSensor()`의 RS485 응답 대기 중 `handleSerial()`을 호출해 USB 시리얼 명령을 처리.
  - 각 센서 항목을 읽는 사이사이에도 `handleSerial()`을 호출해 릴레이 지연을 줄임.

### 3.4 삭제
- 없음.

---

## 4. 배포 후 점검

1. 웹에서 릴레이 ON/OFF 버튼을 눌렀을 때 이전보다 응답 실패가 줄었는지 확인.
2. 페이지를 켜둔 상태에서 30초 간격으로 릴레이 상태가 정상 갱신되는지 확인.
3. 센서값이 다시 약 3초 주기로 들어오는지 확인.
4. 아두이노 USB를 잠깐 재연결하거나 일시 오류가 나도 Pi 쪽이 자동 복구하는지 확인.
