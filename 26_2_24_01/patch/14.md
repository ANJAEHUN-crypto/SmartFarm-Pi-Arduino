# 14차 패치 상세

- **요약**: 센서 통신 오류에서 발생하는 오버플로우/실패값(`65535`, `6553.5`, `-1`)이 저장/표시로 전파되지 않도록, 아두이노·Pi·웹 UI 전 구간에 유효성 검증과 완화 로직을 추가했습니다. 센서 폴링 주기도 30초로 낮춰 RS485 부하를 줄였습니다.

---

## 1. 아두이노 RS485 응답 검증 강화 + 30초 주기 조정

### 1.1 문제
- 응답 프레임 길이만 맞으면 값을 채택하여 깨진 프레임이 `65535` 등 비정상값으로 반영될 수 있었음.
- 센서 송신 주기(3초)가 빠른 환경에서는 통신 여유가 부족해 실패값이 늘어날 수 있었음.

### 1.2 목표
- Modbus 응답이 정상일 때만 센서값을 반영.
- 센서 폴링 부하를 줄여 장시간 안정성 확보.

### 1.3 수정
- **`arduino/smart_farm_relay/smart_farm_relay.ino`**
  - 센서 송신 주기: **3초 -> 30초**.
  - Modbus 응답 검증 추가:
    - 주소/기능코드/바이트수 검증
    - CRC16 검증
  - 센서별 유효 범위 검증 추가:
    - temperature `-10~80`
    - humidity `0~100`
    - EC `0~20000`
    - pH `0~14`
    - N/P/K `0~2000`
  - 유효 범위를 벗어나거나 읽기 실패 시 기존 정상값 유지.

### 1.4 삭제
- 없음.

## 2. Pi 저장/전송 단계에서 비정상 센서값 차단

### 2.1 문제
- 아두이노에서 들어온 raw JSON이 검증 없이 저장/전송되어 비정상값이 히스토리와 MQTT에 그대로 반영될 수 있었음.

### 2.2 목표
- Pi에서도 한 번 더 센서 범위 검증하여 오류값 전파를 차단.

### 2.3 수정
- **`pi/badge_mqtt.py`**
  - 센서 범위 테이블(`SENSOR_RANGES`) 추가.
  - `_sanitize_sensor_payload()` 추가:
    - 숫자 변환 불가/범위 밖 센서 필드 제거
    - 유효 센서 필드가 0개면 해당 프레임 폐기
  - 저장(raw)과 MQTT 퍼블리시 모두 검증된 payload 기준으로 처리.

### 2.4 삭제
- 없음.

## 3. 웹 현재값 표시 안정화 + 조회 주기 완화

### 3.1 문제
- 현재값이 마지막 1개 샘플 기준이라 단일 스파이크에 즉시 흔들릴 수 있었음.
- 배지 조회 주기(5초)가 불필요하게 짧아 트래픽이 높았음.

### 3.2 목표
- 현재값 표시의 순간 튐 완화.
- UI 조회 부하 완화.

### 3.3 수정
- **`pi/static/app.js`**
  - 현재값 계산을 최근 샘플(최대 5개)의 **중앙값(median)** 기반으로 변경.
  - 배지 조회 주기: **5초 -> 30초**.

### 3.4 삭제
- 없음.

## 4. 배포 후 점검

1. 시리얼 로그에서 `65535`, `6553.5`, `-1`, `-0.1` 패턴이 장시간 재발하지 않는지 확인.
2. 웹 현재값이 단일 이상치에 과민 반응하지 않는지 확인.
3. 텔레그램 주기 현황의 센서값이 물리적으로 가능한 범위에서 안정적으로 유지되는지 확인.
4. 릴레이 동작 중에도 센서값이 급격히 깨지지 않는지 확인.
