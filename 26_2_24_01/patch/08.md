# 8차 패치 상세

- **배포**: 8차 적용 완료
- **요약**: 요일별 스케줄 필터(All/Mon~Sun), 영문 요일, 채널당 20개, 24h 시간 표기, **토양 센서 10분마다 송신**

---

## 1. 토양 센서 송신 주기 변경

- **아두이노** `smart_farm_relay.ino`
  - `sensorInterval`: **3000 ms(3초) → 600000 ms(10분)** 으로 변경
  - BADGE JSON 전송이 10분에 1회로 동작
- **의도**: 송신 빈도 감소로 부하·저장량 완화

---

## 2. 스케줄 요일 표현 방식

- **저장/실행**
  - `days`: `daily` 또는 `Mon,Tue,Wed,Thu,Fri,Sat,Sun` 영문 콤마 구분
  - 기존 `0,1,2` 형식도 백엔드(`scheduler_service._should_run_today`)에서 계속 인식
- **프론트**
  - 표시 시 숫자 요일을 영문으로 변환(`formatDays`), 저장 시 영문으로 정규화

---

## 3. 요일별 스케줄 필터(전체/요일별 관람)

- **채널별 요일 필터 버튼**: `All | Mon | Tue | Wed | Thu | Fri | Sat | Sun`
- **동작**
  - `All`: 해당 채널 전체 스케줄 표시
  - 특정 요일: `days`에 해당 요일이 포함된 스케줄만 표시
- **스케줄 추가**: 선택된 요일을 `days` 기본값으로 사용 (All이면 `daily`)

---

## 4. 시간 형식 24시간제(00~24시)

- **formatTime**: `HH:MM` 두 자리로 통일 (예: `09:00`, `23:59`)
- 스케줄 리스트·모달 입력·저장 값에 적용

---

## 5. 채널당 최대 스케줄 20개

- **schedule_store.py**: `MAX_PER_CHANNEL = 20`, 에러 메시지 "Max 20 schedules per channel"
- **app.js**: 스케줄 추가 전 개수 검사 20개로 변경

---

---

## 7. 배포 후 점검

1. 아두이노 재업로드 후 BADGE 전송이 약 10분 간격으로 동작하는지 확인
2. 각 채널에서 All / Mon~Sun 필터로 스케줄이 올바르게 필터되는지 확인
3. 스케줄 추가 시 선택 요일이 기본값으로 들어가는지, 20개까지 추가 가능한지 확인
4. 시간이 `HH:MM` 24시간 형식으로 표시·저장되는지 확인
5. 기존 숫자 요일(0,1,2) 스케줄이 올바른 요일에 실행·표시되는지 확인
