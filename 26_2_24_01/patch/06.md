# 6차 패치 상세

- **배포 시점**: 6차
- **요약**: 웹 그래프 섹션 맨 위 배치, X축 고정·1시간 단위 라벨, N·P·K 센서 그래프 추가

---

## 1. 웹 화면 순서 변경

- **변경**: 그래프(토양 센서 주간 추이) 섹션을 **맨 위**로 이동
- **순서**: 헤더 → **그래프(badge-section)** → 릴레이 → 스케줄
- **파일**: `pi/templates/index.html`

---

## 2. X축 동작 수정

- **고정 범위**: 매 갱신 시 `scales.x.min = 0`, `scales.x.max = 399` 설정 → X축이 끝없이 늘어나지 않음
- **라벨**: **1시간 단위만 표시** — 시각 데이터 중 분(minute)이 0인 시점에만 라벨 출력, 나머지는 빈 문자열
- **파일**: `pi/static/app.js` — `refreshBadgeChart()` 내 라벨 생성 로직 및 `ALL_CHARTS().forEach`에서 x.min/max 설정

---

## 3. N·P·K 그래프 추가

- **데이터**: 센서 JSON에 `soil_N`, `soil_P`, `soil_K` 키가 있으면 수신·저장·표시 (기존 `badge_mqtt`·API는 수정 없이 그대로 사용)
- **그래프**: N / P / K 전용 차트 3개 추가 (총 7개: 온도·습도·EC·pH·N·P·K)
- **Y축**: N·P·K 공통 `0 ~ 200` 고정
- **파일**:
  - `pi/templates/index.html`: `chartN`, `chartP`, `chartK`용 canvas 3개 추가
  - `pi/static/app.js`: `Y_AXIS.npk`, `chartN/P/K` 생성·데이터 바인딩, `soil_N`/`soil_P`/`soil_K` 파싱
